Pick an integration pattern

Every example below is a single HTML file calling the same XPay JavaScript SDK. Open the source for any page to see exactly what's happening.

Drop-in — modal recommended starter

One xpay.checkout({ mode: "modal" }) call opens the full payment UI in an overlay. Card form, 3D Secure, local methods — all inside the iframe.

Use when: you want zero UI work and a "Pay" button.

Drop-in — inline

Same SDK, mounted in a container instead of an overlay. The iframe auto-sizes to its content.

Use when: the form should sit inside the page, not float over it.

Elements — PaymentElement custom UI

Build your own form. Our <PaymentElement> handles the payment method picker and the secure card fields. Call checkout.confirm() with customer details you collected yourself.

Use when: you want full control over layout and styling.

Hosted Checkout — full redirect

Send the customer to checkout.xpay.app for the entire payment, then redirect back. No SDK on your page at all.

Use when: backend-only integration, no JS required.
